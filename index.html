👉 대용량 데이터 집계 쿼리 최적화 (성능 92% 개선)

[Situation] 연간 대시보드 조회 시 월별 쿼리가 반복 실행(N+1)되어 대량 데이터 처리 시 타임아웃 위험 발생.

[Action] GROUP BY를 활용하여 쿼리를 단일화하고, 복잡한 비즈니스 계산은 백엔드에서 Java Stream API로 처리하여 DB 부하 분산.

[Result] 조회 쿼리를 24회에서 2회로 축소, 응답 속도를 2.5초에서 0.4초로 단축.

👉 정산 데이터 정합성을 위한 트랜잭션 설계

[Problem] 전표 생성과 재고 업데이트가 동시에 일어날 때, 부분 실패 시 데이터 불일치 가능성 확인.

[Solution] 서비스 레이어에 @Transactional을 적용하고 고유 번호 생성 로직을 원자적(Atomic)으로 설계하여 정합성 보장.
